boolean insertVex(E v) 插入顶点

将顶点数据 v 添加到顶点数组末尾，顶点数加 1。若已满则返回 false。

public boolean insertVex(E v) {
    if (numOfVexs >= maxNumOfVexs)
        return false;
    vexs[numOfVexs++] = v;
    return true;
}
时间 O(1) 空间 O(1)
boolean insertEdge(int v1, int v2, int weight) 插入边

在 edges[v1][v2] 和 edges[v2][v1] 同时写入 weight，无向图保持对称。

public boolean insertEdge(int v1, int v2, int weight) {
    if (v1 < 0 || v2 < 0 || v1 >= numOfVexs || v2 >= numOfVexs)
        throw new ArrayIndexOutOfBoundsException();
    edges[v1][v2] = weight;
    edges[v2][v1] = weight;   // 无向图：对称赋值
    return true;
}
时间 O(1) 空间 O(1)
boolean deleteVex(E v) 删除顶点

找到顶点 v 的位置 i，前移顶点数组和邻接矩阵的行/列，顶点数减 1。

public boolean deleteVex(E v) {
    for (int i = 0; i < numOfVexs; i++) {
        if (vexs[i].equals(v)) {
            // 前移顶点数组
            for (int j = i; j < numOfVexs - 1; j++)
                vexs[j] = vexs[j + 1];
            vexs[numOfVexs - 1] = null;
            // 压缩矩阵行
            for (int row = i; row < numOfVexs - 1; row++)
                for (int col = 0; col < numOfVexs; col++)
                    edges[row][col] = edges[row+1][col];
            // 压缩矩阵列
            for (int col = i; col < numOfVexs - 1; col++)
                for (int row = 0; row < numOfVexs; row++)
                    edges[row][col] = edges[row][col + 1];
            numOfVexs--;
            return true;
        }
    }
    return false;
}
时间 O(V²) 空间 O(1)
int getEdge(int v1, int v2) 查询边

直接返回 edges[v1][v2]，0 表示无边，其他为权值。

public int getEdge(int v1, int v2) {
    if (v1 < 0 || v2 < 0 || v1 >= numOfVexs || v2 >= numOfVexs)
        throw new ArrayIndexOutOfBoundsException();
    return edges[v1][v2];
}
时间 O(1)
📌 GraphAdjList：每个顶点维护一条边链表（ENode），插入边用头插法，空间 O(V+E)，适合稀疏图。
boolean insertEdge(int v1, int v2, int weight) 插入边（头插法）

为 v1 和 v2 各新建一个 ENode，分别头插到对方的邻接链表。

public boolean insertEdge(int v1, int v2, int weight) {
    if (v1 < 0 || v2 < 0 || v1 >= numOfVexs || v2 >= numOfVexs)
        throw new ArrayIndexOutOfBoundsException();
    ENode vex1 = new ENode(v2, weight);
    if (vexs[v1].firstadj == null) {
        vexs[v1].firstadj = vex1;
    } else {
        vex1.nextadj = vexs[v1].firstadj;  // 头插
        vexs[v1].firstadj = vex1;
    }
    ENode vex2 = new ENode(v1, weight);
    if (vexs[v2].firstadj == null) {
        vexs[v2].firstadj = vex2;
    } else {
        vex2.nextadj = vexs[v2].firstadj;
        vexs[v2].firstadj = vex2;
    }
    return true;
}
时间 O(1)
DFS / BFS（邻接表版） 遍历

遍历邻接链表代替遍历矩阵行，时间复杂度从 O(V²) 降至 O(V+E)。

// DFS 邻接表版（核心差异：遍历链表而非矩阵行）
public String DFS(int v, boolean[] visited, StringBuilder sb) {
    visited[v] = true;
    sb.append(vexs[v].data + ",");
    ENode current = vexs[v].firstadj;   // 遍历邻接链表
    while (current != null) {
        if (!visited[current.adjvex])
            DFS(current.adjvex, visited, sb);
        current = current.nextadj;
    }
    return sb.toString();
}
int getEdge(int v1, int v2)（邻接表版） 查询边

需遍历 v1 的邻接链表找到 adjvex == v2 的结点。

public int getEdge(int v1, int v2) {
    ENode current = vexs[v1].firstadj;
    while (current != null) {
        if (current.adjvex == v2)
            return current.weight;
        current = current.nextadj;
    }
    return 0;  // 无此边
}
时间 O(度数)
📌 IGraph<E> 是图的统一接口，定义了11个方法，邻接矩阵和邻接表都实现该接口。
public interface IGraph<E> {
    int     getNumOfVertex();                     // 获取顶点数量
    boolean insertVex(E v);                      // 插入顶点
    boolean insertEdge(int v1, int v2, int weight); // 插入边
    boolean deleteVex(E v);                      // 删除顶点
    boolean deleteEdge(int v1, int v2);          // 删除边
    int     indexOfVex(E v);                     // 定位顶点（返回索引）
    E       valueOfVex(int v);                    // 查找顶点（返回值）
    int     getEdge(int v1, int v2);             // 查找边（返回权值）
    String  depthFirstSearch(int v);             // 深度优先遍历
    String  breadthFirstSearch(int v);           // 广度优先遍历

}

图的存储结构

📊 邻接矩阵（顺序存储）

用二维数组 edges[i][j] 存储边的权值：

  • edges[i][j] = w：i 到 j 有权为 w 的边
  • edges[i][j] = 0：无边（或对角线）
  • 无向图：edges[i][j] == edges[j][i]（对称矩阵）
A~F 邻接矩阵（部分）
空间 O(V²) 适合稠密图

🔗 邻接表（链式存储）

顶点数组 + 边链表的组合结构：

  • vexs[i].data：顶点数据
  • vexs[i].firstadj：邻接链表头指针
  • 每个 ENode 含 adjvex、weight、nextadj
A 的邻接链表示意
空间 O(V+E) 适合稀疏图

深度优先搜索（DFS）
🔑 口诀：一条路走到黑，走不通再回头。类似"走迷宫"——沿一条路一直走，遇死路退回找下一条路。

算法步骤

1
从起点 v 出发，标记为已访问，加入结果序列
2
遍历 v 的所有邻接点，找到第一个未访问的邻接点 w
3
递归对 w 执行 DFS（沿着这条路深入）
4
当 w 的所有邻接点都已访问，回溯到 v，继续寻找 v 的下一个未访问邻接点
5
重复 2~4 直到所有顶点被访问

以 A-F 交通网为例（从 A 出发）

DFS 遍历结果：A → B → C → E → D → F
路径：A 先走到 B（最小索引邻接点），B 到 C，C 到 E，E 到 D，D 已无未访问邻接点，回溯到 C，C 到 F。

复杂度与应用场景

时间复杂度

  • 邻接矩阵：O(V²)（检查所有矩阵格）
  • 邻接表：O(V+E)（只访问实际边）

应用场景

  • 连通分量检测
  • 环检测（图中是否有环）
  • 拓扑排序（有向无环图）
  • 迷宫求解、路径搜索
广度优先搜索（BFS）
🔑 口诀：先把周围邻居全部访问，再逐层向外扩展。类似"波纹扩散"——从中心向外一圈一圈访问。

算法步骤

1
从起点 v 出发，标记已访问，加入队列 Queue
2
从队列出队一个顶点 u，加入结果序列
3
遍历 u 的所有未访问邻接点，标记并全部入队
4
重复步骤 2~3 直到队列为空

以 A-F 交通网为例（从 A 出发）

BFS 遍历结果：A → B → D → F → C → E
过程：A 入队 → 出队A，B/D/F 入队 → 出队B，C 入队 → 出队D，E 入队 → 出队F → 出队C → 出队E。
💡 BFS 可以求无权图的最短路径（即经过边数最少的路径）。对于带权图的最短路径，需要 Dijkstra 算法。

DFS vs BFS 对比

对比维度DFS 深度优先BFS 广度优先
数据结构栈（递归调用栈）队列
遍历方式纵向深入，回溯横向扩展，逐层
最短路径✗ 不保证✓ 无权图保证
适合问题连通性、环检测最短跳数、层次遍历
内存消耗取决于递归深度取决于同层节点数
Dijkstra 最短路径算法
🔑 问题：从城市 A 出发，求到各城市的最短行车距离。这是单源最短路径问题，Dijkstra 算法用贪心策略求解。

核心思想

维护两个集合：S（已确定最短路径的顶点集） 和 T（未确定的顶点集）。

1
初始化：distance[源点] = 0，其余 = 直接相邻权值（无边则为 ∞），S = {源点}
2
从 T 中选出 distance 最小的顶点 Vᵢ，将其加入 S（此时 distance[Vᵢ] 已是最短距离）
3
松弛操作：对所有 T 中顶点 Vⱼ，若 distance[Vᵢ] + edge(Vᵢ,Vⱼ) < distance[Vⱼ]，则更新 distance[Vⱼ]
4
重复 2~3 直到 T 为空

以 A-F 交通网为例（从 A 出发）

初始化S={A}090∞150∞180
①选 B(90)，S加B090150140∞180

⚠️ 注意事项：
• Dijkstra 只适用于边权非负的图（有负权边时失效）
• 时间复杂度：O(V²) 朴素实现；用优先队列可优化到 O((V+E)logV)
• 邻接矩阵版会修改原始 edges 数组（将0替换为 MAX_VALUE），注意数据保护
