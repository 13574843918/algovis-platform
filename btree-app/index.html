🤔 为什么需要压缩？

  • 存储空间有限，文件越小越好
  • 网络传输带宽宝贵，小文件传得快
  • ASCII每字符固定8位，浪费空间
  • 常用字符可以用更短的编码表示

💡 核心思想

  • 出现次数多的字符 → 编码短
  • 出现次数少的字符 → 编码长
  • 任何字符的编码都不是另一个字符编码的前缀（前缀码）
  • 使用二叉树来组织所有字符的编码

🌳 二叉树的作用

  • 每个字符对应一个叶结点
  • 从根到叶的路径 = 该字符编码
  • 向左走 = 0，向右走 = 1
  • 最优二叉树使总编码长度最小

哈夫曼算法原理

每次选取权值最小的两棵树，合并为新树，新根权值 = 两子树权值之和。重复直到只剩一棵树。

编码规则

从根到每个叶结点的路径即为该字符的编码：
向左走 → 追加 0
向右走 → 追加 1

前缀码保证

叶结点的编码路径不会是其他叶结点路径的前缀，因此解码时不会产生歧义。

public void create(String message) { // 1. 统计字符频率 Map<Character, Integer> charSet = new HashMap<>(); for (char c : message.toCharArray()) charSet.merge(c, 1, Integer::sum); // 2. 初始化数组（2n-1个结点） data = new HNode[charSet.size() * 2 - 1]; // 3. n-1次合并，每次选最小两棵 for (int i = 0; i < leafNum - 1; i++) { // 找x1(最小), x2(次小) data[leafNum+i].setLchild(x1); data[leafNum+i].setRchild(x2); } }

private int preOrder(HNode root, String code) { if (root != null) { root.getData().setCode(code); if (isLeaf(root)) { // 叶结点：记录编码，返回WPL贡献 return weight * code.length(); } // 向左走追加"0"，向右走追加"1" sum += preOrder(data[lchild], code+"0"); sum += preOrder(data[rchild], code+"1"); } return sum; }

🔒 文本编码 codes()

遍历原文每个字符，在哈夫曼树中查找其编码，拼接得到压缩后的二进制串。

public String codes(String message) { StringBuilder sb = new StringBuilder(); HNode root = getRoot(); for (int i = 0; i < message.length(); i++) { char c = message.charAt(i); HNode hn = search(root, c+""); sb.append(hn.getData().getCode()); } return sb.toString(); }

🔓 解码还原 decodes()

从编码串开头逐位匹配，用层次遍历找到匹配的叶结点，输出字符后重新从根开始。

public String decodes(String codes) { Queue<HNode> q = new LinkedList<>(); q.add(getRoot()); String str = ""; while (!q.isEmpty()) { HNode tmp = q.poll(); if (!codes.startsWith(tmp.code)) continue; if (isLeaf(tmp)) { str += tmp.name; // 找到字符！ codes = codes.substring(tmp.code.length()); if (codes.length() > 0) { q.clear(); q.add(root); } } // 继续入队左右孩子 } return str; }

🔍 查找结点 search()

先序递归遍历树，按字符名称查找对应结点，用于编码时获取字符的编码串。

public HNode search(HNode p, String name) { if (p == null) return null; if (p.getData().getName().equals(name)) return p; // 找到！ HNode res = search(data[p.getLchild()], name); if (res != null) return res; return search(data[p.getRchild()], name); }

🌿 链式版：使用PriorityQueue

LinkedHuffmanTree使用Java优先队列（最小堆）自动维护顺序，代码更简洁优雅。

// 创建比较器：按权值→名称长度→字典序排列 PriorityQueue<Node<HData>> queue = new PriorityQueue<>((n1, n2) -> { int x = n1.weight - n2.weight; if (x != 0) return x; return n1.name.compareTo(n2.name); }); // 合并：每次取出两个最小结点 while (queue.size() > 1) { Node lc = queue.poll(), rc = queue.poll(); Node parent = new Node(lc.w+rc.w, lc, rc); queue.offer(parent); }

一、树的基本概念

定义

树（tree）是 n（n≥0）个相同类型的数据元素组成的具有层次关系的集合。n=0 时为空树。对于非空树：有且仅有一个根结点（无前驱）；其余结点被分成 m（m>0）个互不相交的子树。

二、二叉树

定义

二叉树（Binary Tree）由 n（n≥0）个有限结点组成，或者为空，或者由一个根结点及两棵不相交的左子树和右子树的二叉树组成。二叉树最大度为2，是有序树，左右子树不可互换。

5种基本形态

(a)空树 (b)只有根 (c)左单子树 (d)右单子树 (e)完整形态

(a) 空树

空二叉树（n=0），无任何结点

(b) 只有根

只含一个根结点（n=1），无左右子树

(c) 左单子树

有根结点，只有左子树，无右子树

(d) 右单子树

有根结点，只有右子树，无左子树

(e) 完整形态

有根结点，左右子树均存在

5个重要性质

完全二叉树示例：按层编号1~7，对应数组下标0~6

性质1

非空二叉树第 i 层最多有 2^(i-1) 个结点（i≥1）

性质2

深度为 k 的二叉树最多有 2^k - 1 个结点

性质3

叶结点数 n₀ 和度为2的结点数 n₂ 满足：n₀ = n₂ + 1（推论：n个叶结点的哈夫曼树共 2n-1 个结点，无度为1的结点）

性质4

n个结点的完全二叉树深度为 ⌊log₂n⌋ + 1

性质5

完全二叉树编号（从0开始）：序号i的结点双亲为 (i-1)/2，左孩子 2i+1，右孩子 2i+2

四、带权路径长度（WPL）与哈夫曼树

WPL定义

WPL = Σ(Wₖ × Lₖ)，其中 Wₖ 为第k个叶结点的权值，Lₖ 为其路径长度（根到该叶结点的边数）。WPL最小的二叉树称为最优二叉树（哈夫曼树）。

最优二叉树可以有多种形态，WPL值相同且最小。

哈夫曼算法4步

1

由n个权值 {W₁,W₂,…,Wn} 构造n棵只有一个叶结点的二叉树，得到集合 F

2

在F中选取根结点权值最小和次小的两棵树作为左右子树，构造新树，新根权值 = 左权 + 右权

3

从F中删除用掉的两棵树，将新树加入F

4

重复步骤2-3，当F中只剩一棵树时，即为最优二叉树

总编码长度 = 4×1+4×1+4×1+4×1+4×1+4×2+3×2+3×2+3×3+3×3+3×4 = 70位（vs ASCII的168位）

前缀码保证唯一解码

前缀码（Prefix-free Code）：任何字符的编码都不是另一个字符编码的前缀。哈夫曼编码天然满足此性质（编码对应叶结点路径）。解码时每次从根开始匹配，遇到叶结点即输出字符，无歧义。

历史背景

哈夫曼编码由美国计算机科学家 David A. Huffman 于1952年在MIT攻读博士期间发明，发表于论文《A Method for the Construction of Minimum-Redundancy Codes》。这是最优变长前缀码，对给定的字符频率分布，不存在期望编码长度更短的变长前缀码。

五、二叉树的两种存储结构

顺序存储（静态链表）

用一维数组存储所有结点，每个HNode包含数据域 + 左孩子/右孩子/父结点的数组下标（-1表示无）。数组大小为 2n-1（n为叶结点数）。

public class HNode { private int lchild, rchild, parent; // 数组下标，-1表示无 private HData data; // 结点数据 public HNode() { lchild=rchild=parent=-1; } } // 数组布局：[0..n-1]存叶结点，[n..2n-2]存内部结点 HNode[] data = new HNode[2*n-1];

链式存储（二叉链表）

每个结点含数据域 + 左孩子指针 + 右孩子指针，LinkedHuffmanTree使用Java的PriorityQueue维护最小堆。

public class Node<E> { private E data; // 结点数据 private Node<E> lchild; // 左孩子指针 private Node<E> rchild; // 右孩子指针 }

两种方式对比

对比项顺序存储（SeqHuffmanTree）链式存储（LinkedHuffmanTree）
数据结构静态数组 HNode[]动态链表 Node<HData>
构建方式手动查找最小两个PriorityQueue自动维护
代码复杂度较高（嵌套循环找最小）较低（offer/poll即可）
空间分配预先分配2n-1个结点动态分配
