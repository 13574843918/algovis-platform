📖 队列：排队的艺术

队列是一种简单却强大的数据结构，了解它，你会发现它无处不在。

🔷 什么是队列（Queue）？

队列是一种先进先出（FIFO - First In First Out）的线性数据结构。想象银行排队：最早来的人，最先被服务。

队头（front）出队
→
① 小明
② 小红
③ 小刚
→
队尾（rear）入队

核心操作：

• enqueue - 入队，放到队尾

• dequeue - 出队，从队头取出

• peek - 查看队头元素

• 栈像叠盘子，最后放的先拿

• 队列像银行排队，先来的先办业务

➡️ enqueue —— 入队

将一个元素添加到队列的队尾。这是队列最基本的操作。

// 入队操作 public boolean enqueue(E item) { if (isFull()) return false; rear = (rear + 1) % maxsize; data[rear] = item; return true; }
时间复杂度：O(1) — 只需在队尾添加元素。
循环队列注意：队尾指针使用取模运算实现循环。

⬅️ dequeue —— 出队

从队列的队头移除并返回元素。只能从队头删除，不能从中间删除。

// 出队操作 public E dequeue() { if (isEmpty()) return null; front = (front + 1) % maxsize; return data[front]; }
时间复杂度：O(1)
异常情况：对空队列执行 dequeue 会返回 null。

👁️ peek —— 查看队头

返回队头元素的值，但不删除它。用于查看下一位等待的顾客。

// 取队头元素 public E peek() { if (!isEmpty()) { return data[(front + 1) % maxsize]; } else return null; }
时间复杂度：O(1)
peek 与 dequeue 的区别：peek 只读，dequeue 读并删除。

❓ isEmpty —— 判断是否为空

检查队列中是否没有元素，是判断是否有顾客等待的关键。

// 判断队列是否为空 public boolean isEmpty() { return front == rear; } // 循环队列：队空条件 front == rear
时间复杂度：O(1)
银行场景中，isEmpty 返回 true 意味着没有顾客等待，服务窗口可以休息。

⚠️ isFull —— 判断是否为满

检查队列是否已满，防止溢出。

// 判断队列是否为满（循环队列） public boolean isFull() { return (rear + 1) % maxsize == front; } // 队满条件：牺牲一个存储单元
时间复杂度：O(1)
循环队列特点：为了区分队空和队满，牺牲一个存储单元。

// Java 循环顺序队列 SeqQueue public class SeqQueue<E> implements IQueue<E> { private int maxsize; // 队列的容量 private E[] data; // 存储数据元素 private int front; // 队头指示器 private int rear; // 队尾指示器 public SeqQueue(Class<E> type, int maxsize) { data = (E[])Array.newInstance(type, maxsize); this.maxsize = maxsize; front = rear = -1; } public boolean isEmpty() { return front == rear; } public boolean isFull() { return (rear + 1) % maxsize == front; } public boolean enqueue(E item) { if (!isFull()) { rear = (rear + 1) % maxsize; data[rear] = item; return true; } return false; } public E dequeue() { if (!isEmpty()) { front = (front + 1) % maxsize; return data[front]; } return null; } public E peek() { if (!isEmpty()) return data[(front + 1) % maxsize]; return null; } public int size() { return (rear - front + maxsize) % maxsize; } }
// Java 链队列 LinkQueue public class LinkQueue<E> implements IQueue<E> { private QueueNode<E> front; // 队头指针 private QueueNode<E> rear; // 队尾指针 private int maxsize; // 容量限制 private int size; // 元素个数 public LinkQueue() { front = rear = null; size = 0; maxsize = 0; } public boolean isEmpty() { return (front == rear) && (size == 0); } public boolean enqueue(E item) { QueueNode<E> newnode = new QueueNode<>(item); if (!isFull()) { if (isEmpty()) { front = newnode; rear = newnode; } else { rear.setNext(newnode); rear = newnode; } ++size; return true; } return false; } public E dequeue() { if (isEmpty()) return null; QueueNode<E> node = front; front = front.getNext(); if (front == null) rear = null; --size; return node.getData(); } public E peek() { if (!isEmpty()) return front.getData(); return null; } public int size() { return size; } }

什么是队列？

队列是一种线性数据结构，遵循先进先出（FIFO）原则，像银行排队——最早来的最先被服务。

循环队列解决假溢出

普通顺序队列多次入队出队后，rear到达数组末尾无法继续入队，形成假溢出。

循环队列将数组首尾相接，用取模运算实现：

rear = (rear + 1) % maxsize front = (front + 1) % maxsize

✓ 队空：front == rear
✓ 队满：(rear + 1) % maxsize == front

链队列 vs 顺序队列

  • 顺序队列：用数组存储，需预设容量，可能假溢出
  • 链队列：用链表存储，动态扩展，无溢出问题
  • 两者入队、出队时间复杂度都是 O(1)

银行场景中，客流量可预估时用顺序队列；客流量不确定时用链队列。

银行排队系统架构

// 排队机类 - 生产者 public class QueueMachine implements Runnable { private int number; private IQueue<Integer> queue; public void run() { while (true) { scanner.nextLine(); // 等待回车 queue.enqueue(++number); } } } // 服务窗口类 - 消费者 public class ServiceWindow extends Thread { public void run() { while (true) { synchronized(queue) { if (queue.size() > 0) { num = queue.dequeue(); print("请" + num + "号到窗口"); } } sleep(10000); // 服务10秒 } } }
