■ 串的定义

串是由 0个或多个字符组成的有限序列，记作 s = "a₀a₁…aₙ₋₁"。字符个数 n 称为串长，n=0 时称为空串。

  • 子串：任意连续字符的子序列
  • 模式匹配：在主串中查找模式串
  • 串相等：长度相同且对应字符相同

■ IString 接口（ADT）

  • length() — 返回串长度
  • concat(t) — 联接两串
  • compareTo(t) — 字典序比较
  • substring(b,e) — 取子串
  • indexOf(t,b) — 模式匹配定位
  • charAt(i) — 取指定位字符
  • insert(t,off) — 插入子串
  • delete(b,e) — 删除子串

■ 两种存储结构

顺序串 SeqString：char[] 数组，连续内存，O(1) 随机访问，存储效率高。

链串 LinkString：单链表，离散节点，O(1) 插删（已知位置），动态扩展。

■ 模式匹配算法

BF 算法：逐字符匹配，最坏 O(n×m)，实现简单。

KMP 算法：利用 next 数组避免主串回退，时间复杂度优化为 O(n+m)。

串的基本操作详解
length()

获取串长度

返回串中字符的个数。顺序串遍历数组统计非空字符，时间 O(n)；链串遍历链表节点计数。

public int length() { int len = 0; for(int i=0; i<data.length; i++) if(data[i] != '\u0000') len++; return len; }
示例：s = "ababcabda"
s.length() = 9
concat(t)

联接操作

将串 t 联接到当前串末尾，返回新串。顺序串新建数组复制两段字符；链串将 t 的头节点接在末尾。

public IString concat(IString t) { char[] buf = new char[slen+tlen]; // 复制 this → buf[0..slen-1] // 复制 t → buf[slen..slen+tlen-1] return new SeqString(buf); }
s.concat(t) where t="abd"
"ababcabda" + "abd" = "ababcabdaabd"
compareTo(t)

字典序比较

逐字符按 Unicode 值比较。若前缀相同，长串更大；若某位字符不同，返回差值。

for(int i=0; i<lim; i++) { if(v1[i] != v2[i]) return v1[i] - v2[i]; } return slen - tlen;
s="ababcabda", t="abd"
compareTo → 'a'-'d' = -3 (s < t)
substring(b,e)

取子串

提取 [begin, end) 范围内的字符构成新串。需检查下标合法性，时间 O(e-b)。

char[] buf = new char[end-begin]; for(int i=0; i<subLen; i++) buf[i] = data[begin+i]; return new SeqString(buf);
s.substring(3, 7)
"ababcabda"[3..6] = "bcab"
indexOf(t,b)

BF 模式匹配

从 begin 位置开始，用暴力（Brute-Force）逐字符匹配模式串 t。不匹配时 i 回退，j 重置为0。

while(i<len && j<tlen) { if(data[i] == t1.data[j]) { i++; j++; } else { i = i-j+1; j=0; } // 回退 } return j>=tlen ? i-tlen : -1;
s.indexOf(t, 0)，t="abd"
第6次比对成功，返回 5
insert(t,off)

插入子串

在 offset 位置前插入串 t。新数组分三段：offset前、t、offset后。时间 O(n)。

// 新数组 = [0..off-1] + t + [off..n-1] char[] buf = new char[slen+tlen]; // 段1: 复制 [0, offset) // 段2: 复制 t // 段3: 复制 [offset, slen)
s.insert("XY", 5)
"ababc" + "XY" + "abda" = "ababcXYabda"
delete(b,e)

删除子串

删除 [begin, end) 范围的字符，新数组 = begin前段 + end后段。时间 O(n)。

char[] buf = new char[n-(end-begin)]; // 复制 [0, begin) → buf // 复制 [end, n) → buf[begin..] return new SeqString(buf);
s.delete(3, 5)
"aba" + "abda" = "abaabda"
charAt(i)

随机访问字符

顺序串直接下标访问，O(1)；链串需从头遍历到第 i 个节点，O(n)。这是两种结构的关键差异。

// 顺序串：O(1) return data[index]; // 链串：O(n) for(int i=0; i<index; i++) p = p.next; return p.data;
s.charAt(2)
"ababcabda"[2] = 'a'

■ 串的定义与基本概念

串是由 0个或多个字符组成的有限序列，记作 s = "a₀a₁…aₙ₋₁"（n ≥ 0）。n 称为串的长度，n=0 时为空串。

重要特征

串是线性表的特例：数据元素是单个字符（不同于一般线性表元素可以是多字段记录）。

相关术语

  • 子串：串中任意连续字符组成的子序列（含空串）
  • 主串：包含子串的更长的串
  • 字符位置：字符在串中的下标（从0开始）
  • 子串位置：子串的第一个字符在主串中的位置
  • 串相等：两串长度相同且所有对应位置字符完全相同
  • 空格串：由空格组成的串，长度≥1（不同于空串）

ADT 串（IString 接口）

public interface IString { int length(); // 返回串长 IString concat(IString t); // 联接 int compareTo(IString t); // 字典序比较 IString substring(int begin, int end); // 取子串[begin,end) int indexOf(IString t, int begin); // 定位 char charAt(int index); // 取值 IString insert(IString t, int offset); // 插入 IString delete(int start, int end); // 删除 }

■ 顺序串（SeqString）

用 字符数组（char[]）存储所有字符，字符按顺序存放在连续内存中。

存储示意（"STUDENT"）

索引: 0 1 2 3 4 5 6
字符: S T U D E N T

类定义

public class SeqString implements IString { public char[] data; // 字符数组 public SeqString(char[] data) { this.data = new char[data.length]; for(int i=0; i<data.length; i++) this.data[i] = data[i]; } }

特性总结

  • 随机访问：O(1)，直接下标 data[i]
  • 插入/删除：O(n)，需创建新数组（不可变）
  • 存储效率高：每字符只占 1~2 字节，无额外指针
  • 适用：读多写少、需要频繁随机访问的场景

■ 链串（LinkString）

用单链表存储字符，每个节点包含一个字符（data）和指向下一节点的指针（next）。

节点结构示意（"abd"）

start → [a|→] → [b|→] → [d|null]

类定义

public class LinkString implements IString { private class Node { char data; Node next; public Node(char d, Node n) { data=d; next=n; } } private Node start; // 头指针 }

indexOf 三指针法

Node sp = start; // 主串当前比较指针 Node sp0 = sp; // 本轮起始位置 Node tp = t.start; // 模式串指针 while(sp!=null && tp!=null) { if(sp.data == tp.data) { sp=sp.next; tp=tp.next; } else { sp0=sp0.next; sp=sp0; tp=t.start; } }

特性总结

  • 随机访问：O(n)，需从头遍历
  • 插入/删除（已知位置）：O(1)，调整指针即可
  • 存储效率低：每字符需额外一个指针（约50%用于指针）
  • 适用：频繁增删、串长度变化大的场景

块链串（扩展）

为提高链串存储效率，每个节点可存储多个字符（块大小 k），存储密度提升到 k/(k+1)，平衡了存储效率和操作效率。

■ BF（Brute-Force）算法

从主串第 begin 位置开始，与模式串逐字符匹配；遇到不匹配时，主串指针 i 回退重新开始，模式串指针 j 重置为 0。

匹配过程演示（s="ababcabda", t="abd"）

主串 s: a b a b c a b d a
模式串 t: a b d

第1次: s[0..2] vs t[0..2]: a=a ✓ b=b ✓ a≠d ✗ 失败
第2次: s[1..3] vs t[0..2]: b≠a ✗ 失败
第3次: s[2..4] vs t[0..2]: a=a ✓ b=b ✓ c≠d ✗ 失败
第4次: s[3..5] vs t[0..2]: b≠a ✗ 失败
第5次: s[4..6] vs t[0..2]: c≠a ✗ 失败
第6次: s[5..7] vs t[0..2]: a=a ✓ b=b ✓ d=d ✓ 成功！返回 5

复杂度分析

  • 最好情况：O(n)（每次第一字符就不匹配）
  • 最坏情况：O(n×m)（如 s="aaaaab"，t="aaab"）
  • 平均情况：O(n+m)

优缺点

  • ✅ 实现简单，代码直观
  • ✅ 对一般随机文本效率可接受
  • ❌ 存在大量回退，最坏情况慢

■ KMP 算法

利用匹配失败时已匹配部分的信息，避免主串指针回退，只移动模式串指针。时间复杂度 O(n+m)。

关键概念：next 数组（失败函数）

next[j] 表示模式串 t[0..j] 中，最长真前缀 = 最长真后缀的长度。即满足 t[0..k-1] = t[j-k+1..j] 的最大 k 值。

模式串 "ABABCABAB" 的 next 数组：
j: 0 1 2 3 4 5 6 7 8
t: A B A B C A B A B
next: 0 0 1 2 0 1 2 3 4

buildNext 算法

static int[] buildNext(String pat) { int[] next = new int[pat.length()]; int k = 0; for(int i=1; i<pat.length(); i++) { while(k>0 && pat.charAt(i)!=pat.charAt(k)) k = next[k-1]; // 回退 if(pat.charAt(i)==pat.charAt(k)) k++; next[i] = k; } return next; }

KMP 搜索过程

int j = 0; for(int i=0; i<n; i++) { while(j>0 && text.charAt(i)!=pat.charAt(j)) j = next[j-1]; // j 回退，i 不动 if(text.charAt(i)==pat.charAt(j)) j++; if(j==m) return i-m+1; // 匹配成功 } return -1;

BF vs KMP 对比

■ Java 内置串实现

String（不可变串）

JDK 8 及之前：底层使用 char[] 存储，每次修改都创建新对象。JDK 9+：改为 byte[] + coder 字段（Compact Strings），ASCII 字符只占 1 字节，节省约50%内存。

// JDK 9+ String 内部结构 private final byte[] value; // 字节数组 private final byte coder; // LATIN1=0, UTF16=1

三类串的对比

类可变性线程安全性能适用
String不可变安全常量池优化常量、键值
StringBuilder可变不安全最快单线程拼接
StringBuffer可变安全较慢（同步锁）多线程拼接

字符串驻留（Interning）

JVM 维护字符串常量池，字面量字符串自动入池。通过 intern() 可手动驻留，相同内容的字符串共享同一对象，节省内存。

String a = "hello"; String b = "hello"; System.out.println(a == b); // true（常量池共享） String c = new String("hello"); System.out.println(a == c); // false（堆中新对象） System.out.println(a == c.intern()); // true（驻留后同一池对象）
